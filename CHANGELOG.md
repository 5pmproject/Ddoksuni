# 변경 이력

## v1.3.1 (2026-01-27) - 요양병원 vs 요양시설 차이 가이드 추가

### 사용자 혼동 해소
많은 사용자가 **요양병원**과 **요양시설**을 혼동하는 문제를 해결하기 위한 상세 가이드를 추가했습니다.

1. **환자 등록 폼에 눈에 띄는 안내 박스 추가**
   - 장기요양등급 입력 필드 바로 위에 주황색 경고 박스 배치
   - 두 기관의 핵심 차이를 한눈에 비교
   - "상세 차이점 보기 (필독!)" 버튼으로 상세 모달 연결

2. **상세 비교 모달 (`showHelp('facility_types')`)**
   - **요양병원 (의료기관)**:
     - 장기요양등급 필요 없음
     - 건강보험 적용
     - 의사 진료 제공 (의료행위 가능)
     - 산정특례 적용 가능 (암/중증질환 → 5% or 10% 부담)
     - 입원 조건: 의사 판단, 지속적 치료 필요
     - 비용 예시: 폐암 산정특례 시 5% 부담
   
   - **요양시설/요양원 (장기요양기관)**:
     - 장기요양등급 필수 (1~5등급)
     - 장기요양보험 적용
     - 요양보호사 중심 돌봄
     - 의료행위 제한적 (간단한 투약 정도)
     - 이용 조건: 65세 이상 or 노인성 질병
     - 국가 지원 80-85%

3. **실제 경로 비교 예시**
   - 폐암 환자의 전원 경로 시각화:
     ```
     급성기 병원 (수술, 항암)
       ↓
     요양병원 (산정특례만 있으면 됨)
       ↓
     재가 or 요양시설 (장기요양등급 필요)
     ```

4. **실무 팁 제공**
   - 의사의 지속적 치료 필요 → 요양병원 (산정특례)
   - 일상생활 돌봄만 필요 → 요양시설 (장기요양등급)
   - 순차적 이용 가능 (요양병원 → 요양시설)

### 개선 효과
- 이름이 비슷해서 생기는 혼동 해소
- 필요한 제도(건강보험 vs 장기요양보험) 명확화
- 입원/이용 조건의 차이 이해
- 올바른 전원 경로 선택 지원

## v1.3 (2026-01-27) - UX/UI 대폭 개선: 환자 중심 플로우

### 사용자 경험 개선
1. **첫 화면 최적화**
   - 환자 등록 폼을 첫 화면에 최우선 배치
   - 환자 대시보드 제거 (단순한 플로우)
   - 화려한 웰컴 메시지로 사용자 안심감 제공
   - 4단계 프로세스를 시각적으로 명확하게 안내

2. **명확한 단계별 안내**
   - 2단계(경로 추천): 큰 "다음" 버튼으로 비용 계산 유도
   - 3단계(비용 계산): 기관 찾기로 자연스럽게 이동
   - 4단계(기관 찾기): 체크리스트로 마무리
   - 각 단계마다 컬러풀한 안내 박스 추가

3. **자동 플로우 구현**
   - 환자 등록 완료 후 1.5초 후 자동으로 경로 추천 화면으로 이동
   - 웰컴 메시지 자동 숨김
   - 부드러운 스크롤 애니메이션

4. **버튼 디자인 개선**
   - 그라데이션 배경 적용 (단계별 색상 구분)
   - 더 큰 폰트 및 명확한 아이콘
   - 주요 액션(다음)과 보조 액션(선택사항) 구분
   - 호버 효과 강화

5. **웰컴 메시지 강화**
   - 더 넓은 패딩과 그라데이션 배경
   - 4단계 프로세스를 2x2 그리드로 시각화
   - 각 단계별 색상 코드 적용
   - animate-bounce 효과로 주목도 향상

### 주요 변경사항
- **src/index.tsx**: 각 단계별 안내 박스 및 버튼 레이아웃 개선
- **public/static/app.js**: 자동 플로우 로직 구현

### 사용자 혜택
- 처음 방문한 사용자도 쉽게 시작 가능
- 다음에 무엇을 해야 할지 명확하게 안내
- 불필요한 정보 제거로 집중도 향상
- 전체 프로세스를 한눈에 파악 가능

## v1.2 (2026-01-27) - Glasgow Coma Scale (GCS) 의식 상태 평가 도입

### 주요 변경사항
1. **의식수준 평가 방식 개선**
   - 단순 선택 방식 (명료/기면/혼미/혼수) → 표준화된 GCS 평가로 전환
   - 세 가지 영역 독립 평가:
     - 눈 뜨기 반응 (Eye Opening): 1-4점
     - 언어 반응 (Verbal Response): 1-5점
     - 운동 반응 (Motor Response): 1-6점
   - 총점 3-15점으로 객관적 의식 수준 평가

2. **실시간 GCS 점수 계산**
   - 세 영역 선택 시 자동으로 총점 계산
   - 중증도별 색상 코드:
     - 15점: 정상 (초록)
     - 13-14점: 경미한 장애 (파랑)
     - 9-12점: 중등도 장애 (노랑)
     - ≤8점: 중증 장애 (빨강)

3. **상세 GCS 평가 가이드**
   - 각 영역별 점수 기준 상세 설명
   - 실제 환자 상태 예시 제공
   - 지남력, 주의력, 인지기능 설명
   - 평가 팁 및 주의사항
   - 의료진 확인 방법 안내

4. **데이터베이스 스키마 업데이트**
   - GCS 점수 저장 필드 추가 (gcs_total, gcs_eye, gcs_verbal, gcs_motor)
   - 기존 환자 데이터 자동 변환
   - GCS 검색 인덱스 추가

### 의학적 정확성 향상
- 국제 표준 의식 평가 도구 적용
- 의료진과의 커뮤니케이션 용이
- 재활 계획 수립에 더 정확한 데이터 제공
- 섬망(Delirium), 혼란(Confusion) 상태 구분 가능

### 사용자 경험
- 진료기록지의 GCS 점수 그대로 입력 가능
- 각 항목별 명확한 설명으로 정확한 평가 가능
- 의료진에게 문의할 내용 명확화

## v1.1 (2026-01-27) - 사용자 친화적 입력 가이드 추가

### 개선사항
1. **ADL 점수 입력 개선**
   - 숫자 입력 → 슬라이더 UI로 변경
   - 실시간 점수 표시
   - 상세 평가 가이드 모달 추가
   - 간단 평가 방법 제공 (5가지 항목 체크)
   - 중증도별 상세 설명 (0-30점/40-60점/70-100점)

2. **건강보험 유형 설명**
   - 각 옵션에 상세 설명 추가

   - 확인 방법 안내

3. **장기요양등급 안내**
   - 각 등급별 상세 설명 추가 (1급-5급)
   - 점수 기준 표시
   - 신청 방법 3단계 안내
   - '아직 신청 안함' 옵션으로 명확화
   - 혜택 정보 제공

4. **발병일 입력 가이드**
   - 질환별 기준일 설명
   - 뇌졸중/암/골절 등 예시 제공
